Every failure raised by the SDK is an instance of `PoliPageError` or one of its subclasses. The `code` attribute tells you what went wrong; the `request_id` attribute is what to quote when filing a support ticket.

## The error hierarchy

`PoliPageError` is the base catch-all. Two transport-level branches (`APIConnectionError` and its `APITimeoutError` subclass) and one HTTP-status branch (`APIStatusError`, with per-status subclasses like `RateLimitError`, `NotFoundError`, `BadRequestError`) cover every failure mode.

```python
from poli_page import PoliPage, PoliPageError

client = PoliPage()

try:
    client.render.pdf({"project": "billing", "template": "invoice", "data": {}})
except PoliPageError as err:
    print(err.code, err.request_id, err.message)
```

## Example

```python
from poli_page import PoliPage, PoliPageError, RateLimitError

client = PoliPage()

try:
    pdf = client.render.pdf({
        "project": "billing",
        "template": "invoice",
        "data": {"invoiceNumber": "INV-001", "total": 1280},
    })
except RateLimitError:
    # Custom strategy beyond the SDK's built-in retries.
    pdf = None
except PoliPageError as err:
    print(f"render failed: {err.code} ({err.request_id})")
    raise
```
